## 1. 프로젝트 개요

**Kcode**는 한글 명령어로 작성하는 프로그래밍 언어입니다.

### 주요 목표

- **한글 문법** — 모든 키워드를 한글로 작성, 명령어는 가급적 3자 이내
- **이중 실행 방식** — 인터프리터(바로 실행)와 컴파일러(실행파일 생성) 모두 지원
- **이미지 인식 + 렌더링 특화** — OpenCV(이미지), OpenGL(렌더링) 내장 래핑
- **LLM API 연동** — ChatGPT, Claude, Gemini, 로컬 모델 연결 지원
- **GC(가비지 컬렉터) 내장** — 포인터 없이 메모리 자동 관리

### 타겟 사용 분야

```
이미지 처리 프로그램
렌더링 / 게임 / 시각화
AI 비전 애플리케이션 (이미지 인식 + LLM 결합)
```

### 참고 언어

|특징    |참고 언어           |
|------|----------------|
|문법 구조 |Python (들여쓰기 기반)|
|컴파일 방식|LLVM 백엔드 활용     |
|객체지향  |Java / Kotlin   |
|전처리기  |C / C++         |

## 2. 핵심 설계 철학

### 포인터 없음

요즘 언어 트렌드에 맞게 포인터를 제거하고 GC로 메모리 자동 관리

### 함수와 정의 구분

반환값 유무로 명확히 구분하여 코드 가독성 향상

```
함수 더하기(정수 가, 정수 나):    // 반환값 있음 (function)
    반환 가 + 나

정의 인사(문자 이름):             // 반환값 없음 (void/sub)
    출력("안녕하세요, " + 이름)
```

### 사진과 그림 구분

이미지 관련 자료형을 목적에 따라 명확히 구분

```
사진  - 파일에서 불러온 이미지    사진열기("photo.jpg")
그림  - 코드로 직접 그리는 캔버스 그림만들기(800, 600)
```

### 접두어 없는 내장 함수

`수학.제곱근()` 대신 `제곱근()` 으로 바로 사용

### 3자 이내 키워드

가능한 한 짧게 — `최댓값` → `최대`, `난수정수` → `난정수`

## 7. 함수 체계

### 함수 (반환값 있음)

```
함수 더하기(정수 가, 정수 나):
    반환 가 + 나
```

### 정의 (반환값 없음)

```
정의 인사(글자 이름):
    출력("안녕하세요, " + 이름)

정의 검사(정수 값):
    만약 값 < 0:
        끝냄          // 중간 탈출
    출력("양수: " + 값)
```

### 기본값 매개변수

```
함수 인사(글자 이름 = "손님"):
    반환 "안녕하세요, " + 이름

인사()            // "안녕하세요, 손님"
인사("홍길동")    // "안녕하세요, 홍길동"
```

### 가변 매개변수

```
함수 합계(정수... 수들):
    정수 합 = 0
    각각 수 안에 수들:
        합 = 합 + 수
    반환 합

합계(1, 2, 3)         // 6
합계(1, 2, 3, 4, 5)   // 15
```

### 다중 반환값

```
함수 최대최소(배열 수들):
    반환 최대(수들), 최소(수들)

정수 큰값, 정수 작은값 = 최대최소([3, 1, 5, 2])
```

### 람다 (익명 함수)

```
함수형 제곱 = (정수 x) => x * x
출력(제곱(5))    // 25

배열 숫자들 = [1, 2, 3, 4, 5]
배열 제곱들 = 숫자들.변환((정수 x) => x * x)
배열 짝수들 = 숫자들.걸러내기((정수 x) => x % 2 == 0)
```

## 8. 자료구조 체계

### 배열 (array)

```
배열 과일 = ["사과", "배", "감"]
과일[0]
과일.추가("포도")
과일.제거("배")
과일.길이()
과일.정렬()
과일.걸러내기((글자 x) => 길이(x) > 1)
과일.변환((글자 x) => 대문자(x))
```

### 사전 (dictionary)

```
사전 사람 = {"이름": "홍길동", "나이": 30}
사람["이름"]
사람["주소"] = "서울"
```

### 목록 (struct - 데이터 묶음)

```
목록 좌표:
    실수 x
    실수 y

좌표 위치 = 좌표(1.0, 2.5)
위치.x
위치.y
```

### 객체 (class)

```
객체 사람:
    글자 이름
    정수 나이

    함수 생성(자신, 글자 이름, 정수 나이):
        자신.이름 = 이름
        자신.나이 = 나이

    정의 소개(자신):
        출력(자신.이름 + " / " + 자신.나이 + "살")

// 상속
객체 학생 이어받기 사람:
    글자 학교

    함수 생성(자신, 글자 이름, 정수 나이, 글자 학교):
        부모.생성(이름, 나이)
        자신.학교 = 학교
```

### 열거형 (enum)

```
열거 방향:
    위
    아래
    왼쪽
    오른쪽

방향 이동방향 = 방향.위
```

### 인터페이스 (interface)

```
틀 도형:
    함수 넓이()
    함수 둘레()
    정의 정보출력()

객체 원 이어받기 도형:
    // 반드시 넓이(), 둘레(), 정보출력() 구현해야 함
    // 미구현시 컴파일 오류
```

### 색상 함수 (별도 가져오기 불필요)

```
// 생성
RGB(빨강, 초록, 파랑)
RGBA(빨강, 초록, 파랑, 투명도)
HSL(색조, 채도, 밝기)
16진색("#FF5733")

// 상수
빨강  초록  파랑  노랑  주황  보라

// 조작
밝기(색, 양)              // -100 ~ 100
```

## 11. 컴파일러 구조

```
.han 소스코드
      ↓
  [렉서]  한글 키워드 → 토큰
      ↓
  [파서]  토큰 → AST
      ↓
  [인터프리터]  AST 직접 실행 (바로 실행 모드)
      또는
  [코드 생성]  AST → LLVM IR
      ↓
  LLVM 백엔드
      ├── OpenCV  → 사진/이미지 처리
      ├── OpenGL  → 2D/3D 렌더링
      ├── HTTP    → LLM API 연동
      └── CUDA    → GPU 가속
      ↓
  실행파일 (.exe / .apk)
```

### 구현 언어 권장

- **컴파일러 본체** : C 또는 C++ (성능, LLVM 연동)
- **표준 라이브러리** : C로 구현 후 Kcode로 래핑
- **이미지 엔진** : OpenCV 래핑
- **렌더링 엔진** : OpenGL / Vulkan 래핑

## 12. 설계 결정 사항

|항목    |결정           |이유                  |
|------|-------------|--------------------|
|포인터   |**없음**       |요즘 언어 트렌드, GC로 대체   |
|메모리 관리|**GC 내장**    |사용자 편의성             |
|들여쓰기  |**Python 방식**|친숙함                 |
|키워드 길이|**3자 이내 권장** |타이핑 편의성             |
|함수 구분 |**함수 / 정의**  |반환값 유무로 명확 구분       |
|이미지 구분|**사진 / 그림**  |파일 vs 캔버스 명확 구분     |
|내장 함수 |**접두어 없음**   |`수학.제곱근()` → `제곱근()`|
|LLM   |**API 연동**   |직접 구현 대신 외부 연결      |
|챗봇 기능 |**제외**       |시장 포화, 차별화 불필요      |
|구조체 이름|**목록**       |struct 개념           |
|배열 이름 |**배열**       |사람들에게 익숙한 용어        |
